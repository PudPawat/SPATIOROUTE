SpatioRoute: Dynamic Prompt Routing for Zero-Shot Spatial Reasoning

SpatioRoute is a zero-shot framework that routes each spatial question to a semantically tailored prompt before VLM inference — without fine-tuning, 3D sensors, or point-cloud inputs. Two modes are supported: SpatioRoute-R (rule-based template routing) and SpatioRoute-L (LLM-driven routing with few-shot demonstrations, video-free at routing time). On SQA3D, SpatioRoute improves overall accuracy by up to 5% over fixed-prompt baselines across multiple VLM families.

Motivation

Fixed prompt vs. SpatioRoute. Existing methods apply one generic template to every question (top). SpatioRoute selects or generates a question-type-specific prompt with spatial guidance (bottom), then sends it with video frames to a vision-language model.

Abstract

Spatial question answering over egocentric video is a challenging task that requires vision-language models (VLMs) to reason about 3D object positions, scene affordances, and directional relationships, particularly in the zero-shot setting where no task-specific fine-tuning is available. We introduce SpatioRoute, a dynamic prompt generation approach that routes each incoming question to a semantically tailored prompt template — without any additional training, fine-tuning, or 3D sensor input.

SpatioRoute operates in two complementary modes: SpatioRoute-R, a rule-based router that deterministically maps SQA3D question typologies (What, Is, How, Can, Which) to specialized prompt templates; and SpatioRoute-L, an LLM-driven approach that generates task-specific prompts from the question and situational context alone, with no video input at routing time. We evaluate SpatioRoute on the SQA3D benchmark across VLMs spanning multiple model families. SpatioRoute achieves consistent overall accuracy gains up to 5% over fixed prompt baselines, establishing a new state-of-the-art for zero-shot video-only spatial VQA without requiring 3D point-cloud inputs. As an additional finding, we observe that Chain-of-Thought (CoT) prompting, implemented via the Think-it-Twice architecture, consistently degrades performance in this setting on Qwen series models, confirming that question-aware routing is more effective than uniform reasoning instructions for spatial video understanding.

Method

Overview of SpatioRoute. Inputs are video V, question q, and situational context s. Method I (SpatioRoute-R): question clarification → prompt template library → deterministic mapping (Table 1) → VLM. Method II (SpatioRoute-L): text-only LLM with K=6 few-shot demonstrations (video-free at routing) → VLM. Video frames are consumed only at the final VLM stage.

SpatioRoute-R (Rule-based)

Classifies each question into SQA3D types (What / Is / How many / Can / Which / Others) and fills the corresponding YAML template with situation and question text. Fast, deterministic, and fully reproducible.

SpatioRoute-L (LLM-driven)

Uses the same routing table, then a small text LLM (e.g. Qwen2.5-0.5B) rewrites the prompt using few-shot layout examples. Routing uses question + situation only — no video at this stage.

Key Findings

Condition Description Outcome on SQA3D
Fixed baseline One prompt for all questions Lower overall accuracy

Code & Quick Start

git clone git@github.com:PudPawat/SPATIOROUTE.git
cd SPATIOROUTE
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# SpatioRoute-R: rule-based preprompts
python preprompts/generate_rule.py \
  --split test --output results/preprompts_r_test.json

# SpatioRoute-L: LLM + few-shots
python preprompts/generate_llm.py \
  --split test --output results/preprompts_l_test.json

# VLM evaluation with routed prompts
python eval/routed.py \
  --preprompt-json results/preprompts_l_test.json \
  --model-name Qwen/Qwen2-VL-2B-Instruct --load-in-4bit \
  --output results/spatioroute_l_test.json

See the README for dataset setup, smoke tests, and multi-model evaluation.
